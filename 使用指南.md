# 校园网 IP 自动更新系统 - 使用指南

## 📋 系统概述

本系统实现了校园网 IP 自动检测和推送功能，通过 Gitee Pages 提供访问跳转服务。

### 核心功能

1. **自动 IP 检测**：每 5 分钟检测本机校园网 IP 变化
2. **自动推送**：IP 变化时自动更新到 Gitee 仓库
3. **网络检测**：前端页面自动检测是否在校园网内
4. **智能跳转**：校园网内自动跳转到查寝系统

## ⚙️ 配置说明

### 1. 环境变量（可选）

在 `.env` 文件中配置：

```bash
# Gitee 配置
GITEE_USER=KAERdashu
GITEE_TOKEN=your_personal_access_token
GITEE_REPO=https://gitee.com/KAERdashu/dorm-check.git
```

**获取 Gitee Token 步骤**：
1. 登录 Gitee → 设置 → 个人访问令牌
2. 生成新令牌
3. 勾选权限：`repo`（仓库）
4. 复制并保存 Token（只显示一次）

### 2. 网卡配置

默认配置：
- 网卡名称：`WLAN 2`
- 检测间隔：`300 秒`（5 分钟）

如需修改，编辑 `backend/app.py`：

```python
ip_service = init_ip_update_service(
    repo_path=repo_path,
    check_interval=300,  # 修改检测间隔（秒）
    wlan_interface="WLAN 2"  # 修改网卡名称
)
```

## ⚠️ 常见问题

### Q1: IP 一直是 127.0.0.1 怎么办？

**原因**：获取到了本地回环地址

**解决**：
1. 确认网卡名称正确（`WLAN 2`）
2. 使用 `ipconfig` 查看实际网卡名称
3. 修改 `backend/app.py` 中的 `wlan_interface` 参数

### Q2: Git Push 失败怎么办？

**原因**：认证失败或仓库不存在

**解决**：
1. 确认已在 Gitee 创建仓库
2. 配置 GITEE_TOKEN 环境变量
3. 检查网络连接

### Q3: Gitee Pages 访问 404？

**原因**：Pages 未启用或部署失败

**解决**：
1. 检查仓库是否为公开
2. 重新启用 Gitee Pages
3. 等待 3-5 分钟 CDN 刷新

### Q4: 同学访问提示"无法访问系统"？

**可能原因**：
1. ❌ 不在校园网（用流量）
2. ❌ 校园网设备隔离（不同楼宇）
3. ❌ 后端服务未启动
4. ❌ IP 已过期（还未推送）

**解决**：
1. 确认连接校园网 WiFi
2. 确认在同一网段
3. 检查后端服务状态
4. 手动推送一次 IP
